Description

You're almost there. You already have working buttons that can input numbers. We can't call our app a calculator yet, as it doesn't really calculate. Let's change that!

Objectives
  1. After typing in the number and pressing an operation button (+ - * /) your app should:
    • If displayEditText text is not empty get it's value and save it as first term of the operation
    • If displayEditText text is empty get the hint instead and save the value as first term of the operation
    • Save the operation
    • Set displayEditText hint to the number displayed on text
    • Set displayEditText text to empty string
  2. After pressing the second number and clicking equalButton, your app should:
    • Make the operation using both terms
    • Set displayEditText hint to the result of the operation
    • Set displayEditText text to empty string
  3. Give support for chained operations using the result of previous operation
    • ex: `2 + 3 = + 5 =` results on 10;
  4. Give support for unary minus.
    • If the first term is not set interpret the first click to subtractButton as unary minus for the first term
      • ex1: `- 5 + 8 =` results on 3;
      • ex2: `- 1 * 2 =` results on -2;
    • If the first term is set interpret the first click to subtractButton as subtract operation. After the operation is set interpret click to subtractButton as unary minus for the second term
      • ex1: `8 - 5 =` results on 3;
      • ex2: `8 - - 5 =` results on 13;
      • ex3: `1 * - 2 =` results on -2;
      • ex4: `1 * - 2 = + - 2 =`results on -4;
      • ex5: `1 * - 2 = - - 2 =`results on 0;
